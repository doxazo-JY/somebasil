## 1. 페이지 리디자인 — "각 페이지가 다른 질문에 답한다"

카페가 지속적으로 적자인 상황 → 오너가 점장에게 압박 → 점장이 데이터로 원인 설명할 수 있어야 함.
각 페이지를 특정 질문에 답하는 구조로 재설계.

| 페이지 | 핵심 질문 |
|--------|-----------|
| 대시보드 | 지금 어떤 상황인가? |
| 수입 | 매출은 얼마나 나오고 있나? |
| 지출 | 어디서 돈이 새고 있나? |
| 이익 | 흑자가 되려면 뭐가 바뀌어야 하나? |
| 주간현황 | 이번 주는 어땠나? (주간보고용) |

### 지출 페이지 (`/expenses`)
- 헤더: "지출 — 어디서 돈이 새고 있나?"
- `CostRatioCards`: 매출 대비 비용 비율 카드 (신호등 색상)
  - 인건비율: 정상 <35%, 주의 >35%, 위험 >45%
  - 원가율: 정상 <30%, 주의 >30%, 위험 >40%
  - 고정비율: 정상 <20%, 주의 >20%, 위험 >30%

### 이익 페이지 (`/profit`)
- 헤더: "이익 — 흑자가 되려면?"
- `BreakEvenSection`: 손익분기 분석 (매출 얼마 더 필요 / 지출 얼마 줄여야)
- `DeficitBreakdown`: 비용 구조 분해 (카테고리별 매출 대비 바 차트)
- 좌우 2칸 레이아웃 (`grid grid-cols-1 lg:grid-cols-2`)

## 5. 주간현황 탭 신설 (`/weekly`)

### 배경
- 점장이 매주 주간보고를 해야 하는 상황
- 주 단위로 매출/지출/손익을 한눈에 볼 수 있는 화면 필요

### 구현
- 사이드바 메뉴 순서 변경: 대시보드 → **주간현황** → 이익 → 수입 → 지출 → 직원 → 업로드 → 설정
- `app/weekly/page.tsx` 신설
- `components/dashboard/WeeklySection.tsx` 신설
  - 이번 주 요약 카드 4개 (매출 / 지출 / 손익 / 기간)
  - 전주比 증감률 뱃지 (▲▼)
  - 최근 8주 라인차트 (매출 실선 / 지출 점선)
  - 주차별 테이블 (이번 주 강조)
- `lib/supabase/queries/weekly.ts` 신설
  - 월요일 기준 주 시작/종료일 계산
  - `daily_sales` + `monthly_expenses` 주별 집계
  - `WeekData` 인터페이스 export

### 주차 레이블 포맷
- 변경 전: `3/2주`, `4/20주`
- 변경 후: `3월 첫째 주`, `4월 셋째 주`
- 계산: `Math.ceil(날짜 / 7)` → 첫째/둘째/셋째/넷째/다섯째

## 6. 매출 데이터 소스 구조 설계 (핵심)

### 배경
- 현재: POS 데이터 없음, 통장 거래내역만 있음
- 미래: POS 데이터가 생기면 함께 사용할 예정
- 문제: 통장 카드사 입금 ≈ POS 매출 (같은 금액) → 이중 계산 위험

### 결정: Option B — source 컬럼으로 우선순위 관리

#### DB 변경 (직접 실행)
```sql
ALTER TABLE daily_sales ADD COLUMN source TEXT DEFAULT 'bank';
```

#### 데이터 흐름
| 업로드 타입 | 저장 위치 | source 값 |
|------------|-----------|-----------|
| 통장 수입 | `daily_sales` (날짜별 합산, category='income') | `'bank'` |
| POS 매출 | `daily_sales` (카테고리별) | `'pos'` |
| 통장 지출 | `monthly_expenses` | — |

#### 우선순위 로직
- 같은 날짜에 `pos`와 `bank` 데이터가 모두 있으면 → `pos` 사용
- `bank`만 있으면 → `bank` 사용
- `pos`만 있으면 → `pos` 사용

#### 적용 위치
- `app/api/upload/save/route.ts` — 업로드 시 source 태그
- `lib/supabase/queries/weekly.ts` — 주간 집계 시 POS 우선
- POS 업로드 시 `monthly_summary.income`도 재집계
