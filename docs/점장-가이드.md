# 카페 썸바실 관리 시스템 — 사용 가이드

## 한 눈에 보는 구조

사이드바는 **3개 그룹**으로 나뉩니다.

| 그룹 | 언제 봐요 | 페이지 |
|---|---|---|
| **운영** | 매일·매주 | 대시보드 / 주간 |
| **결산** | 월말·분기 | 손익 / 지출 / 메뉴 |
| **관리** | 데이터 입력·정리 | 업로드 / 직원 / 설정 |

페이지 위쪽 작은 탭으로 같은 그룹 안에서 이동할 수 있어요. 그룹 이동 시 보고 있던 월(예: 3월)은 자동으로 따라갑니다.

---

## 🟢 운영 그룹

### 1. 대시보드 (`/`)

> **매일 아침 첫 화면.** "지금 카페 상황이 어떤가" 한눈에.

#### 첫 줄 요약 (가장 먼저 보세요)
한 줄에 4가지가 압축돼 있어요:
- **부분월 표시** — "4월 27일까지 · 월말까지 3일"
- **손익분기 갭** — "⚠ 손익분기까지 30만 부족" 또는 "✓ 손익분기 +20만 초과"
- **어제 매출** — "어제 18만 ▲5.2% 전주同曜比" (현재 월 보고 있을 때만)
- **가장 큰 신호** — 인건비율·재료비율·고정비율 중 전월比 많이 늘어난 항목을 빨강·주황으로 표시. 클릭하면 상세 페이지로

#### KPI 카드 (4개)
이번 달 매출 / 지출 / 순이익 / 객단가. **카드 클릭하면** 해당 상세 페이지로 점프.

#### 적자 원인 신호
인건비율 / 재료비율 / 원가율(준비 중) / 고정비율 — 매출 대비 비율과 전월比 변화를 색깔로.

#### 매출 달력
일별 매출 히트맵. **빨간 점**이 찍힌 날 = 영업일 평균보다 40% 이상 적게 팔린 날. 원인 추적해보세요 (날씨, 이벤트, 데이터 누락 등).

#### BEST 10 메뉴
이번 달 잘 팔린 메뉴 + 전월 순위 변동(▲▼).

#### 메모
이번 달 한 줄 코멘트. **여기 적어두면 손익 페이지(/profit) 월별 표에서도 보여요.** 사장 미팅 전 미리 적어두세요.

#### 12개월 트렌드 + 전체 누적
역사적 맥락 참고용.

#### 💡 팁
- 우상단 월 선택으로 과거 월도 볼 수 있어요
- 매출 달력 빨간 점이 가장 빠른 이상 신호

---

### 2. 주간 (`/weekly`)

> **매주 목요일 주간보고 준비용.** 기본값 = 지난 주.

#### 첫 줄 요약
- 주 라벨 + 일수 (부분주면 "부분주" 표시)
- 손익분기 갭
- 가장 큰 전주比 악화 (KPI 5개 중)

#### 주간 KPI 5개
매출 / 지출 / 손익 / 객단가 / 주문 건수 — 각 카드에 전주 값 + 변화율.

#### 일별 매출 차트
요일별 막대 + 전주 오버레이. "이번 주 화요일이 평소보다 약했다" 같은 패턴 즉시 인지.

#### 카테고리 믹스 + BEST 10
전주比 변화 같이 표시.

#### 💡 팁
- 우상단 주 선택으로 최근 8주 비교
- 부분주(주 중간이면) = 전주도 같은 요일까지만 자동 컷 → 공정 비교

---

## 🟡 결산 그룹

### 3. 손익 (`/profit`)

> **사장 미팅 메인 화면.** 연간 흐름 + 월별 코멘트.

#### YTD 누적 KPI
올해 누적 매출 / 지출 / 순이익.

#### 월평균 손익분기 분석
"매출을 X% 늘리거나 / 지출을 Y% 줄이면 흑자".

#### 12개월 추이 + 월별 손익 표
- 표의 각 월 아래에 그 달 메모(대시보드에서 적은 거)가 같이 보여요
- **월 클릭하면** 그 달 대시보드로 점프 → 메모 편집 가능

#### 💡 팁
- 사장 노트북 미팅 시 메인으로 보여주는 페이지
- 메모는 미리 대시보드에서 적어두세요. 월별 표에서 "1월 왜 적자?" 즉답 가능

---

### 4. 지출 (`/expenses`)

> **"어디서 돈이 새고 있나?"** 의 답.

#### 첫 줄 요약 (액션 인사이트)
- **전월比 가장 많이 증가한 카테고리** — 예: "⚠ 전월比 재료비 +50만 (250만 → 300만, +20%)". 즉시 액션 포인트
- **가장 큰 지출** — 보통 인건비

#### 매출 대비 비용 비율
인건비 / 재료비 / 고정비 비율 카드 (원가율은 메뉴 원가 등록 80%+ 도달 후 활성).

#### 카테고리별 추이
12개월 라인 차트 (인건비·재료비(현금)·재료비(카드)·고정비·설비투자).

#### 거래처별 누적 (사장 보고용)
홍인호·김인성·한성욱·소금집 4공급처 이번 달 + 연간.

#### 항목별 상세
카테고리 안에서 어떤 거래 있었는지 펼쳐 보기.

#### 💡 팁
- 첫 줄 빨간 메시지가 매월 핵심
- 분류가 잘못된 거래는 **업로드 페이지 → 통장 거래내역 탭 → 거래 재분류**에서 수정 (PC만)

---

### 5. 메뉴 분석 (`/menu`)

> **분기 메뉴 정리할 때 + 가끔 죽은 메뉴 청소.**

#### 첫 줄 요약
- 활성 메뉴 X개 / 한 번도 안 팔린 Y개 / 기간 내 0건 Z개

#### 요일×시간 히트맵
"화요일 오후 2시가 가장 한산하다" 같은 패턴 → 인력 배치·이벤트 의사결정.

#### 죽은 메뉴 리스트
- 마지막 판매 오래된 순 (한 번도 안 팔린 게 최우선)
- 위쪽 selector로 기간 변경 (전체 / 4주 / 8주 / 12주 / 이번 달)

#### 죽은 메뉴 행 옆 두 버튼
- **✏️** = 메뉴 매칭 (POS 이름이 마스터와 다를 때)
- **🗑** = 비활성화 (이 메뉴 사용 안 함 마킹)

> 비활성화 = **삭제 아님**. 매출 데이터는 그대로 살아있고, 단지 "이 마스터는 분석 대상 아님" 표시.

#### 💡 팁
- 빨간 점(0건) 메뉴는 한 번도 안 팔린 거. 단종 후보
- 신메뉴 출시 후 깜빡 잊고 안 팔린 것처럼 보일 수도 → 매칭 확인

## 🔵 관리 그룹

### 6. 업로드 (`/upload`)

#### 일별 매출 탭
POS에서 받은 일일 매출 엑셀 업로드. 매일 또는 매주.

#### 통장 거래내역 탭
하나은행 엑셀 업로드.
- 자동으로 인건비 / 재료비(현금) / 재료비(카드) / 고정비 / 설비투자 분류
- **거래 재분류** 표(PC만): 자동 분류가 잘못됐으면 카테고리 드롭다운으로 수정

#### 메뉴 마스터 탭
POS 메뉴 마스터 엑셀 업로드 (분기 1회 또는 신메뉴 출시 시).
- **메뉴 마스터 관리** 표(PC만): 마스터 청소 작업

#### 메뉴 마스터 관리 표 (PC만)
- **1열**: 마스터명 (정렬 토글)
- **2열**: 매칭된 POS 이름 chips (초록=수동 / 회색=자동) + ✏️ 편집
- **3열**: 활성/비활성 토글

**필터** 두 가지:
- "비활성 포함" — 비활성화한 메뉴도 같이 봄
- **"정리 필요만 보기"** — chip이 "—"인 행만. 청소 시 이거 체크하면 1/10로 줄음

**행 라벨**:
- ⚠ 키 충돌 — 다른 활성 마스터와 겹침. 한쪽 비활성화 권장
- ◌ 매칭 없음 — POS에 안 팔리는 메뉴

## 📦 데이터가 어떻게 흘러가는지

```
POS 일일매출 엑셀  →  실제 판매 거래 (어떤 메뉴 얼마 팔렸나)
POS 메뉴 마스터    →  메뉴 카탈로그 (분석 대상 정의)
하나은행 엑셀      →  지출 자동 분류

자동 매칭 (시스템) → POS 거래와 마스터 연결
수동 매칭 (점장)   → 자동으로 안 잡히는 케이스
마스터 청소 (점장) → 중복/단종 비활성화

→ 모든 분석은 마스터 + 거래 매칭 결과 기준
```

**중요한 한 줄**: 비활성화 = 삭제 아님. POS 거래 데이터는 절대 안 건드림. 단지 "어느 마스터로 분류하나"만 정리.

## 📅 자주 하는 작업

### 매일 아침 (1분)
1. 대시보드 열기
2. 첫 줄 요약 한 줄 보기
3. 매출 달력 빨간 점 있으면 원인 추적

### 매주 (5분)
1. 통장 엑셀 다운로드 → 업로드 페이지 → 통장 탭
2. 자동 분류 결과 살짝 확인 (재분류 표에서 잘못된 거 수정)

### 매주 목요일 (주간보고 준비, 10분)
1. 주간 페이지
2. 첫 줄 요약 + 전주比 변화 확인
3. BEST 10 순위 변동 보기

### 매월 결산 (사장 미팅 전, 30분)
1. 대시보드 메모에 한 줄 코멘트 ("3월: 신메뉴 출시로 인건비 ↑")
2. 손익 페이지 열어서 월별 표 + 메모 확인
3. 사장 미팅 시 손익 페이지 보여주기

### 분기 (메뉴 정리, 30분~1시간)
1. POS 메뉴 마스터 신규 export
2. 업로드 페이지 → 메뉴 마스터 탭 → 파일 업로드
3. 메뉴 마스터 관리 표에서 **"정리 필요만 보기"** 체크
4. ⚠ 또는 ◌ 행 처리:
   - 단종/중복 → 비활성화
   - 이름이 POS와 다름 → ✏️ 매칭

## 📱 폰 vs PC

| 작업 | 폰 | PC |
|---|---|---|
| 대시보드·주간·손익·지출·메뉴 보기 | ✅ | ✅ |
| 메모 입력 | ✅ | ✅ |
| 파일 업로드 | ✅ | ✅ |
| 거래 재분류 | ❌ | ✅ |
| 메뉴 마스터 관리 | ❌ | ✅ |
| 설정 | ✅ | ✅ |

> 폰은 콕핏 (보는 용도), PC는 관리 (정리 용도).

## 🛠️ 추후 추가 예정

- **메뉴 원가 계산** — 레시피 등록 후 메뉴별 마진 분석
- **직원/인건비 관리** — 회계사 데이터 정리 후 재설계
- **신호등 절대 임계값** — 운영 데이터 6개월 축적 후 (예: 인건비율 35% 위험 등)

## 헷갈리면 한 줄 룰

- **chip "—"인 활성 마스터** = 정리 후보 (보통 비활성화)
- **비활성화 ≠ 삭제** = 데이터 손실 없음
- **자동 매칭 잘 되면** ✏️ 안 눌러도 OK
- **사장 미팅 전** = 대시보드 메모 한 줄 + 손익 페이지

문의 사항 있으면 알려주세요!
